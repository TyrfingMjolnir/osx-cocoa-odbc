OdbcAppDelegate Class Reference

Inherits from NSObject
Conforms to NSApplicationDelegate
Declared in OdbcAppDelegate.h

Overview

The OdbcAppDelegate class is inteded to use instead of XCode generated AppDelegate code when creating a Core Data project.

When creating a new application, uncheck ‘Use Core Data’ check box. Then let your AppDelegate inherit from OdbcAppDelegate like this:


// Appdelegate.h

#import <Cocoa/Cocoa.h>
#import <Odbc/Odbc.h>

@interface AppDelegate : OdbcAppDelegate 

@property (assign) IBOutlet NSWindow * window;

@end

In your now empty AppDelegate.m implement following method:



// AppDelegate.m

- (NSURL *) persistentStoreUrl {

    return [NSURL URLWithString : @"odbc:///testdb?username=root&password=secret"];
}

This method will be called by OdncAppDelegate to get ODBC database URL. The URL shown above is only an example. You should replace ‘testdb’ by your own ODBC data source name (DSN). You should replace ‘root’ with your username and you should replace ‘secrect’ by your own password.

persistentStoreUrl

This property returns ODBC URL. It should be overwriten by subclasses.

@property (readonly, nonatomic) NSURL *persistentStoreUrl

Return Value

ODBC URL for ODBC database to use.

Discussion

Returned URL should have following format:


odbc::///[data source name]/?username=[username]&password=[password]

where |data source name] is ODBC data source name, [username] is database username and [password] is users password.
